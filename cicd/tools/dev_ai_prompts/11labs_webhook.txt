You are an expert Python backend engineer and Nylas Scheduler API integrator.

Goal:
Generate a fully working Python Flask app with routes that integrate with the Nylas v3 Scheduler API:
- API docs: https://developer.nylas.com/docs/api/v3/scheduler/

The Flask app must:

1) Use environment variables for config
   - NYLAS_API_KEY
   - NYLAS_API_BASE_URL (default: https://api.us.nylas.com if not set)
   - NYLAS_SCHEDULER_ID or any required identifier (e.g. configuration_id, booking_calendar_id, session_id, etc. — use whatever is appropriate based on the Scheduler docs)
   - Optional: PORT

2) Dependencies
   - flask
   - requests
   - python-dotenv (optional, just show how to load .env in development)

3) Implement a route to RECEIVE a webhook-style JSON request to create a booking

   - Route: POST /api/schedule
   - Content-Type: application/json
   - Request body example:
     {
       "date": "2025-11-20",           // YYYY-MM-DD
       "start_time": "13:00",         // 24h format, local time of organizer
       "duration_hours": 4,           // integer, 4-hour block
       "timezone": "America/Los_Angeles"  // IANA tz string
     }

   Behavior:
   - Validate input (date, time, duration_hours, timezone).
   - Convert the date + time + timezone into the appropriate timestamps for the Nylas Scheduler Availability request. Use the correct time format per the docs.
   - Define the time window as a 4-hour range starting at the requested start_time on that date.
     For example:
       requested start = 13:00
       requested end   = 17:00  (start + duration_hours)
   - Call the appropriate Nylas v3 Scheduler availability endpoint to check if this 4-hour block is free for the organizer’s calendar.
     - Use Nylas session / configuration / scheduler ID as required by the docs.
     - Use a reasonable meeting duration (e.g., 240 minutes) and interval as needed, based on the availability API requirements.
   - Inspect the response:
     - If the requested 4-hour window is fully available (no conflicts), then:
       - Call the Nylas v3 Scheduler booking endpoint to create a booking for that exact time range.
       - Include any required fields (title, participant email, organizer, etc.) using placeholders or sample values if necessary.
       - Return HTTP 201 with JSON:
         {
           "status": "success",
           "message": "Slot booked",
           "booking_id": "<nylas_booking_id>",
           "start_time": "<ISO or timestamp>",
           "end_time": "<ISO or timestamp>"
         }
     - If the time range is not fully available:
       - Return HTTP 409 with JSON:
         {
           "status": "unavailable",
           "message": "Requested time slot is not available"
         }

   - Include robust error handling:
     - If the Nylas API returns an error, log it and return HTTP 502 with an error message.
     - If the request body is invalid, return HTTP 400 with a useful error description.

       }
